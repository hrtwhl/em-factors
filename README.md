# EM Factor Portfolio — Project Documentation

## Overview

This project screens country-level factors for predictive power in Emerging Markets,
then backtests long-only strategies using the most promising factors. It is split
across two machines: a **university PC** (Bloomberg Terminal) for data collection,
and a **personal laptop** for analysis.

---

## File Inventory

### R Scripts (run on university Bloomberg PC)

| File | Purpose |
|---|---|
| `config.R` | **Shared configuration.** Defines the 15-country universe (MSCI index tickers → country names), all Bloomberg field definitions (19 fundamental fields), macro ticker mappings (FX, CDS, bond yields, policy rates, CPI, current account), and the master registry of all 35 factors with metadata (ID, label, direction, category). Every other R script sources this file. |
| `getIndexData.R` | **Downloads daily price data** for all indices (equity + bonds) from Bloomberg in EUR. Produces `EM_Indices_EUR.csv`. This is your original script — unchanged. |
| `downloadFactors.R` | **Downloads index-level fundamental data** (PE, P/B, ROE, EPS, etc.) monthly from Bloomberg using the MSCI country index tickers. Produces `raw_factor_data.csv` in long format (date, country, ticker, field, value). |
| `downloadMacro.R` | **Downloads macro data** (FX spots, sovereign CDS, 10Y yields, policy rates, CPI, current account) from Bloomberg using **country-specific tickers** (not the MSCI indices). Produces `raw_macro_data.csv` in long format (date, country, series, value). |
| `prepareForBacktest.R` | **Transforms raw data into Python-ready CSVs.** Reads `EM_Indices_EUR.csv` + `raw_factor_data.csv` + `raw_macro_data.csv`, then: (1) computes monthly returns from daily prices, (2) passes through fundamental factors, (3) computes derived factors (momentum at 1M/3M/6M/12M, volatility, beta, max drawdown, earnings revision, sales revision, FX momentum, real yield, yield curve, CPI change, etc.), (4) writes everything to `backtest_data/` folder. |

### Python Scripts (run on personal laptop)

| File | Purpose |
|---|---|
| `factor_backtest.py` | **Core engine module** (imported by the other two scripts, not run directly). Contains: `BacktestConfig` (settings), `FactorBacktest` (runs the quintile sort), `BacktestResult` (stores results, computes stats, generates charts), and `check_factor_eligibility()` (pre-screens data quality). |
| `run_all_backtests.py` | **Factor screening script.** Phase 1: pre-screens every factor for data eligibility (≥10 countries per date, ≥36 eligible months) — rejects factors with insufficient coverage. Phase 2: runs quintile-sort backtests on all eligible factors. Phase 3: prints comparison tables. Phase 4: generates presentation-quality charts. |
| `strategy_backtest.py` | **Strategy backtest script.** For each selected factor: buys the top bucket (B1 = 3 countries) equal-weighted, applies €81/trade transaction costs on a €60k portfolio, compares to MSCI EM benchmark, tests both monthly and quarterly rebalancing. Outputs net Sharpe, cost drag, holdings composition. |

## Execution Order

### Step 1 — University PC (Bloomberg)

Place in one folder: `config.R`, `getIndexData.R`, `downloadFactors.R`,
`downloadMacro.R`, `prepareForBacktest.R`.

```r
# 1a. Download daily prices (skip if you already have a recent EM_Indices_EUR.csv)
source("getIndexData.R")
# → produces: EM_Indices_EUR.csv

# 1b. Download fundamental factor data (19 fields × 15 indices, monthly)
source("downloadFactors.R")
# → produces: raw_factor_data.csv

# 1c. Download macro data (FX, CDS, yields, rates, CPI, current account)
source("downloadMacro.R")
# → produces: raw_macro_data.csv

# 1d. Convert everything into Python-ready format
source("prepareForBacktest.R")
# → produces: backtest_data/ folder containing:
#      returns.csv
#      factor_trailing_pe.csv, factor_forward_pe.csv, ... (up to 35 files)
#      factor_registry.csv
```

**Copy to your laptop:** the entire `backtest_data/` folder AND `EM_Indices_EUR.csv`.

### Step 2 — Personal Laptop (Python)

Your folder should look like this:

```
em-factors/
├── EM_Indices_EUR.csv            ← from Step 1a

│   ├── returns.csv
```

## What Each Script Actually Computes

### Quintile Sort (run_all_backtests.py)

For each month t:
1. Read the factor value for every country at date t.
2. Read the return for every country at the **next** date after t (month t+1).
3. Inner-join: keep only countries that have **both** a factor value at t and a return at t+1.
4. If fewer than 10 countries survive → skip this month.
5. Sort countries by factor value (ascending or descending depending on direction).
6. Divide into 5 equal buckets: B1 (most attractive) through B5 (least attractive).
7. Compute the equal-weighted return of each bucket over month t+1.
8. Repeat for every month in the dataset.

Output: time series of monthly returns per bucket, from which we compute
annualised return, volatility, Sharpe, max drawdown, monotonicity, and rank IC.

### Strategy Backtest (strategy_backtest.py)

For each month t (or every 3rd month if quarterly):
1. Same ranking as above → identify B1 countries (top 3 out of 15).
2. Compare B1 to current holdings → compute trades (sells + buys).
3. Each trade costs €81 → total cost deducted from portfolio.
4. Hold B1 equal-weighted → collect return over month t+1.
5. In non-rebalancing months (quarterly mode): hold existing portfolio, no trades.
6. Track gross return, net return (after costs), and MSCI EM benchmark.

## Key Parameters

| Parameter | Value | Where set |
|---|---|---|
| Number of buckets | 5 (quintiles) | `factor_backtest.py` default, also in both runner scripts |
| Min countries per period | 10 | `factor_backtest.py` default |
| Min eligible periods | 36 months | `run_all_backtests.py` |
| Portfolio size | €60,000 | `strategy_backtest.py` |
| Cost per trade | €81 | `strategy_backtest.py` |
| Rebalancing | Monthly + Quarterly | `strategy_backtest.py` tests both |
| Benchmark | MSCI EM (MIMUEMRN Index) in EUR | Extracted from `EM_Indices_EUR.csv` |
| Country universe | 15 EM countries | `config.R` |

## The 35 Factors by Category

### Value (9 factors)
Trailing PE, Forward PE, Price-to-Book, Dividend Yield, Price-to-Cash-Flow,
Forward EV/EBITDA, Price-to-Sales, Price-to-Free-Cash-Flow, Forward Dividend Yield

### Quality (5 factors)
Forward ROE, Trailing ROE, Trailing ROA, Profit Margin, Net Debt/EBITDA

### Momentum & Risk (8 factors)
12M Momentum, 6M Momentum, 3M Momentum, 1M Reversal,
12M Realized Volatility, Risk-Adjusted Momentum, Max Drawdown 12M, Beta to EM

### Sentiment (2 factors)
Earnings Revision (3M), Sales Revision (3M)

### Macro (11 factors)
Sovereign CDS 5Y, CDS 3M Change, 10Y Bond Yield, Real 10Y Yield,
Yield Curve, Carry (Policy Rate), FX Momentum 3M, FX Momentum 12M,
CPI YoY, CPI 3M Change, Current Account (% GDP)

**Note:** Many macro factors will be rejected by the eligibility screen due to
insufficient country coverage from Bloomberg. This is expected — the fundamental
and price-derived factors have much better coverage.

## Troubleshooting

- **Bloomberg ticker errors in downloadMacro.R:** ~20% of macro tickers may fail.
  The script prints which ones failed — update `config.R` with correct tickers
  and re-run.
- **"MSCI EM benchmark not found":** The column in `EM_Indices_EUR.csv` may be
  named `MIMUEMRN.Index` (with a dot) instead of `MIMUEMRN Index` (with a space).
  Both are handled automatically.
- **All factors rejected:** If `run_all_backtests.py` rejects everything, check
  that `backtest_data/` contains actual data. Run the diagnostics script from
  earlier to inspect country counts per factor file.
- **Strategy results look bad:** Compare gross vs net returns. If gross is
  reasonable but net is poor, the issue is transaction costs (consider quarterly
  rebalancing). If gross is also poor, the factor signal may not survive in a
  concentrated 3-country portfolio.
